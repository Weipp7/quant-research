## 说明

这是个人量化研究平台,非 Web 应用 — 传统"QA 测试页面"不适用。测试计划聚焦于**回测引擎正确性**和**数据管道完整性**,因为这两者出错会静默污染所有研究结果。

## Critical 测试(Phase 2.5 必须完成才能进 Phase 3)

### Golden Backtest 1: 买入持有

- **文件**:`tests/test_engine.py::test_buy_and_hold_kweichow_moutai`
- **输入**:2020-01-02 用 ¥100,000 全仓买入贵州茅台,持有到 2023-12-31,不调仓
- **验证**:最终 NAV 与 Excel 手算值偏差 < 0.01%
- **手算步骤**:
  1. 查 2020-01-02 收盘价 P0
  2. 查 2023-12-31 收盘价 P1
  3. 查期间所有分红和送转事件
  4. Excel 算出最终 NAV(含现金股利)
- **失败意味着**:回测引擎的除权或 NAV 计算有 bug

### Golden Backtest 2: 月度等权调仓

- **文件**:`tests/test_engine.py::test_monthly_rebalance_equal_weight`
- **输入**:2020-01 至 2020-12,每月第一交易日用全部现金等权买入沪深 300 前 5 只成分股
- **验证**:最终 NAV 与 Excel 手算值偏差 < 0.01%
- **失败意味着**:调仓逻辑或等权计算有 bug

## 架构级测试(融入数据层,非单独跑)

### 防 Lookahead Bias

- **文件**:`tests/test_data_loader.py::test_no_future_data_leak`
- **验证**:`get_bars(symbol, as_of_date, lookback_days)` 返回结果里 `max(trade_date) < as_of_date`(严格小于)
- **失败意味着**:数据 API 会泄漏未来数据,所有回测结果不可信

### 防 Survivorship Bias

- **文件**:`tests/test_constituents.py::test_historical_constituents_not_current`
- **验证**:`load_index_components("000300.SH", date(2020,6,30))` 返回的成分股列表**不等于**当前 `load_index_components("000300.SH", today())`
- **失败意味着**:指数成分股取的是当前快照,不是历史

### 成分股空集保护

- **文件**:`tests/test_constituents.py::test_empty_constituents_raises`
- **验证**:模拟 Tushare 返回空 list 时,函数回溯到前一个有效日期;回溯 7 天仍空 → raise
- **失败意味着**:静默的 Silent Zero bug 可能潜入

## 数据管道测试(Phase 2 完成后)

### 行数一致性

- **文件**:`tests/test_ingest.py::test_ingest_row_count`
- **验证**:拉取后 DuckDB 里 `COUNT(*)` == 预期行数(trading_days × symbols)

### 限流重试生效

- **文件**:`tests/test_ingest.py::test_retry_on_rate_limit`
- **验证**:mock AKshare 第 1 次返回 429,第 2 次成功 — 最终写入成功,无缺失

## 跨平台一致性(Phase 3)

### 本地 vs 聚宽

- **触发**:Phase 3 — 同一个双均线策略在聚宽和本地分别跑
- **验证**:年化收益偏差 < 1%,最大回撤偏差 < 2%
- **失败排查顺序**:
  1. 复权方式(前复权 / 后复权 / 不复权)是否一致
  2. 交易成本(佣金 + 印花税 + 过户费)是否一致
  3. 滑点模型是否一致
  4. 停牌 / 涨跌停 / ST 处理是否一致
- **Golden backtest 已通过后**,这一步差异必定在配置里,不会在引擎里
