## 复权方式

股票分红、送股、配股后股价会出现"跳空"，历史价格需要调整才能连续比较。

| 模式 | 说明 | 适用场景 | 陷阱 |
|------|------|----------|------|
| **不复权 (raw)** | 原始价格，不作任何调整 | 看真实成交价 | 图上有明显跳空，均线等技术指标失真 |
| **前复权 (qfq)** | 历史价格向下调整，最新价 = 真实市价 | 画图、简单策略回测 | 同一段历史数据过段时间再拉，数值可能变（中间又分红了）|
| **后复权 (hfq)** | 历史价格向上调整，以最早日期为基准 | 收益率计算、长期对比 | 最新价远高于真实市价，不能直接用来判断现在"该买多少股" |
| **动态复权** | 聚宽 `use_real_price=True`，每天以当天为基准做前复权 | 聚宽平台内部 | 与本地 qfq/hfq 数据都有细微差异，**是 Phase 3 结果对不上的最常见原因** |

### 复权引起的价差示例
```
000001 2020-01-02 收盘价:
  不复权  ≈ 14.00 元
  前复权  ≈ 13.69 元  (已调整此前所有分红)
  后复权  ≈ 某个更高的值
```

## 手续费

| 项目 | 比例 | 收取时机 | 最低 |
|------|------|----------|------|
| 佣金 (commission) | 万三 (0.03%) | 买入 + 卖出 各一次 | ¥5 / 笔 |
| 印花税 (stamp tax) | 千一 (0.1%) | **仅卖出时** | 无 |
| 过户费 | 极小可忽略 | 卖出时 | — |

聚宽策略代码对应设置：
```python
set_order_cost(OrderCost(
    close_tax=0.001,
    open_commission=0.0003,
    close_commission=0.0003,
    min_commission=5
), type='stock')
```

**本地引擎当前状态**：Phase 2.5 无手续费（用于验证纯逻辑）。Phase 3 加入后 NAV 会低于无手续费版本。

### 手续费对回测的影响估算
- 每次完整交易（买+卖）约 0.13%（万三×2 + 千一）
- 如果策略每月换仓一次，年化交易成本约 0.13% × 12 ≈ 1.6%
- 高频策略成本会很高，这是回测结果"看起来很好、实盘却差很多"的常见原因之一

## 滑点

期望成交价格与实际成交价格之间的偏差。

| 来源 | 说明 |
|------|------|
| 冲击成本 | 大单影响市价，越大的仓位偏差越大 |
| 竞价不确定性 | 开盘竞价拿不到"开盘价"，收盘集合竞价也是 |
| 盘口价差 | 买一卖一之间的 tick 差 |

**回测里的常见处理**：
- 零滑点：假设精确按收盘/开盘价成交（最乐观，结果偏好看）
- 固定滑点：每笔加 N 分钱偏差
- 百分比滑点：价格 × 0.01% 偏差

**本地引擎当前状态**：零滑点。聚宽默认也是零或极小，这一项不是 Phase 3 差异的主因。

## 停牌处理

A 股股票可停牌数天到数月，停牌期间：无价格数据、无法交易。

**停牌对回测的影响**：
- 打算在停牌日买/卖 → 订单应顺延到复牌第一天（或直接取消）
- 若忽略停牌强行用"最近一笔价格"成交 → 引入不存在的流动性，回测失真

**本地引擎当前状态**：未处理（遇到缺数据日期会导致 KeyError）。Phase 2.5 的测试股票（茅台、平安银行、万科）极少停牌，所以没暴露问题。

**高风险场景**：
- 小市值股、ST 股票频繁停牌
- 沪深 300 成分股回测时，某段时间的成分股里可能有临时停牌的

## 生存偏差 (Survivorship Bias)

独立概念，但常与停牌一起踩坑。

回测用"当前的指数成分股"而不是"那个时候的成分股"，会选入所有"活下来"且表现好的公司，高估策略收益。
