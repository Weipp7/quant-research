## 关键认知（按时间线）

### 1. 复权方式很关键
- 回测必须用**后复权**或**不复权+处理事件**
- 前复权数据会随时间变化，导致今天的回测下个月再跑结果不一样
- 平台默认复权方式不一定一致，要主动确认

### 2. 停牌、涨跌停是回测虚假收益的主要来源
- 一字板根本买不到/卖不出，但简陋回测会"假装成交"
- 必须显式检查 `paused` 和 `high_limit/low_limit`
- 这些"幻觉成交"集中在强势/弱势股上，会系统性高估收益

### 3. order_value 有取整问题
- 聚宽 `order_value(stock, amount)` 会向下取整到100股的整数倍
- 应该手动 `int(amount/price/100)*100` 计算手数后用 `order(stock, shares)`
- V1 的 88% 收益里有部分来自这个 bug 导致的"不规范成交"

### 4. ETF 不适合做网格
- ETF 设计初衷就是"降低波动"，但网格依赖波动赚钱
- 即使加动态间距，年化天花板也就 3-5%
- ETF 的优点（分散）不能弥补"波动率被稀释"的缺点

### 5. 仓位管理与网格哲学冲突
- 网格是被动的"逢低买/逢高卖"
- 仓位管理是主动的"判断市场状态"
- 两者前提不同，强行结合反而互相打架
- 测试结果：仓位管理降低了回撤但同时降低了夏普

### 6. 网格策略的真实定位
- **不是 alpha 来源**，而是"震荡放大器"+"分批执行工具"
- 真正的 alpha 来自标的本身的"长期上涨"
- V1 的 88% 主要是茅台、招行、平安等股票 2015-2021 上涨贡献的
- 想突破年化 10% 必须解决"选股"问题 → 多因子

## 已知坑（避免重复踩）

1. **小心"虚高回测"**
   - 完美的回测往往有 bug
   - V1 的高收益部分来自 order_value bug
   - 总要问：实盘真能这样下单吗？

2. **避免过度优化**
   - V2 加了一堆机制（止损、冷静期、趋势过滤、仓位管理），收益反而暴跌
   - 简单的 V1 框架反而最稳健
   - 每加一个机制都要测试是否真的有效

3. **资金量影响策略**
   - 10 万本金限制了高价股（茅台、平安）的纳入
   - 同样的策略在 100 万本金下结果会不同
   - 设计策略时要明确目标资金量

4. **不要混淆机制和效果**
   - "动态间距生效"不等于"提升收益"
   - V1.2 交易笔数确实增加了 21%，但收益只多了 0.5%
   - 要看最终的风险调整收益（夏普），不只是单一指标

## V-Final 的设计哲学

经过这一轮迭代，V-Final 的设计回归本源：

- **简洁**：去掉所有"看起来精妙"的机制
- **稳健**：5股+5ETF分散，不依赖单一标的
- **诚实**：接受网格策略的天花板（年化6-8%）
- **基础**：作为投资组合的"底层配置"，不是收益主力

下一步研究方向应该是**多因子选股**（见 `docs/multifactor_guide.md`），那才是突破10%的可能路径。

## 如何使用这些代码

每个文件都是**完整自包含**的聚宽策略代码，可以直接复制粘贴到聚宽 IDE。

复制流程：
1. 用 `python tools/sync_to_clipboard.py strategies/grid/v_final.py`
2. 到聚宽创建新策略 → 粘贴
3. 设置回测时间区间 → 运行
4. 把结果截图保存到 `results/grid/`
5. 关键指标记录到 `docs/research_log.md`
